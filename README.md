# 每日项目反馈 - 组员修改要点

一个小程序，用来记录每天早晨给组员的项目反馈，按**组员**和**日期**分类，组员可以按日期查看自己需要修改的要点，避免遗漏。

## 功能

- **录入反馈**：选择日期、组员姓名、填写修改要点（可选）、上传反馈图片（可选，可多张）
- **查看反馈**：按日期、组员筛选，列表展示文字与图片
- **导出 / 导入**：导出为 JSON 文件备份或分享；组员可导入你分享的文件查看
- **通过链接查看**：你把导出的 JSON 放到可访问的链接（如 GitHub Gist raw、网盘直链），组员在页面输入链接即可只读查看，无需导入

## 本地运行

直接用浏览器打开 `index.html` 即可，无需安装任何东西。数据保存在本机浏览器（IndexedDB）。

## 让组员也能查看的几种方式

### 方式一：部署到网页，分享链接 + 导出文件

1. 把整个 `daily-feedback-app` 文件夹部署到任意静态托管（如 **GitHub Pages**、**码云 Gitee Pages**、**Vercel**、**Netlify** 等），得到访问地址，例如：`https://你的用户名.github.io/daily-feedback-app/`
2. 你每天在「录入反馈」里记录，在「查看反馈」里点 **导出数据**，把导出的 JSON 文件发到群里（或上传到网盘/ Gist 后发链接）
3. 组员打开上述网页地址，在「查看反馈」里：
   - **导入数据**：选择你发的 JSON 文件，即可看到反馈；或
   - **输入数据链接**：若你把 JSON 放在一个可直接访问的 URL（如 Gist 的 raw 链接），组员输入该链接点「加载」，即可只读查看

### 方式二：仅分享导出文件

不部署网页也可以：你在本机用浏览器打开 `index.html`，录入并导出 JSON，把文件发给组员；组员用浏览器打开同一个 `index.html`，点击「导入数据」选择该文件即可查看。

### 方式三：通过链接只读查看（无需导入）

1. 你导出 JSON 后，将文件内容上传到可公网访问的地址，例如：
   - [GitHub Gist](https://gist.github.com/)：新建 Gist，粘贴 JSON，取 **Raw** 链接
   - 或任何返回 JSON 的直链（需支持浏览器 CORS）
2. 组员打开你部署的页面（或本地 `index.html`），在「输入数据链接」里粘贴该 Raw 链接，点「加载」，即可按日期、组员筛选查看，无需导入文件。

## 文件说明

- `index.html` - 主页面
- `style.css` - 样式
- `app.js` - 逻辑与本地存储（IndexedDB）、导出/导入、远程加载

## 备注

- 数据与图片存在浏览器本地，清除站点数据会丢失；重要内容请定期使用「导出数据」备份。
- 图片以 Base64 形式存在导出文件中，单次导出过多或图片很大时，文件会较大，可考虑按日期或组员分批导出。
